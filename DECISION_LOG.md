# 决策日志 (Decision Log)

每条决策记录格式：
```
### D<编号>: <决策标题>
- **日期**: YYYY-MM-DD
- **背景**: 为什么需要做这个决策
- **选项**: 考虑了哪些方案
- **决策**: 最终选择
- **理由**: 为什么这样选
- **影响**: 对后续工作的影响
- **状态**: active / superseded / reverted
```

### D001: 竞赛技术路线选择
- **日期**: 2026-05-17
- **背景**: AI4S CNS药物发现竞赛，需要从零构建分子生成Agent
- **选项**: (A) 纯RDKit流水线 (B) LLM驱动Agent (C) 混合方案
- **决策**: 方案C：LLM驱动 + RDKit工具辅助
- **理由**: 竞赛评分包含llm_score，纯RDKit方案得0分；纯LLM无法做对接和逆合成
- **影响**: 架构选型确定，后续所有开发基于kimi_agent_sdk
- **状态**: active

### D002: 进化搜索引擎中的分子生成方式
- **日期**: 2026-05-18
- **背景**: evolution.py 中 `_random_mutate()` 用 RDKit 直接变异分子
- **选项**: (A) 保留RDKit变异 (B) 全部改为LLM tool call (C) 混合
- **决策**: 方案B：移除RDKit直接变异，所有分子生成必须经LLM
- **理由**: 2026-05-18提交分数为0，llm_score=0.0，根因是RDKit变异绕过LLM
- **影响**: evolution.py 大幅重写，运行时间增加但保证llm_score>0
- **状态**: active

### D003: 逆合成路线分隔符
- **日期**: 2026-05-20
- **背景**: route_score=0.0，占30%总分全丢
- **选项**: (A) 保持 ` | ` 分隔 (B) 改用 `,` 分隔
- **决策**: 方案B：用 `,` 分隔多步路线
- **理由**: 竞赛评分系统要求 `,` 分隔；` | ` 导致解析失败，route_score=0
- **影响**: synthesis_v2.py 全部路线生成逻辑修改；旧result.csv必须重新生成
- **状态**: active

### D004: 原子平衡验证
- **日期**: 2026-05-20
- **背景**: route_validity_score=0.0，逆合成路线存在化学不可能反应
- **选项**: (A) 修复SMARTS规则 (B) 添加原子平衡验证 (C) 两者都做
- **决策**: 方案C：精简SMARTS规则 + 添加原子平衡验证
- **理由**: 酰胺规则错误匹配导致"两个喹唑啉→一个小分子"(原子凭空消失)；balance_score=0→route_score=0
- **影响**: RETRO_RULES从50+精简为9条经过验证的规则；新增_check_atom_balance()
- **状态**: active

### D005: SA Score计算方式
- **日期**: 2026-05-20
- **背景**: 自研estimate_sa_score()与竞赛SAScore偏差大
- **选项**: (A) 继续用启发式 (B) 用RDKit内置SAScore
- **决策**: 方案B：使用RDKit Contrib的SAScore模块
- **理由**: 启发式公式严重低估SA(2-3 vs 实际4-6)，可能导致SA>6分子漏过滤
- **影响**: evaluator.py 重写estimate_sa_score()；可能影响已有分子的过滤结果
- **状态**: active

- **理由**: 多次提交发现问题无法追溯；修改间互相覆盖；缺乏提交前质量把关
